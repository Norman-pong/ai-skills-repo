# Skill: novel-craft（小说创作与质量管控）

面向中长篇连载小说的标准化生产流程：先规划与索引，再由子代理按统一约束创作章节，最后以量化评分与一致性检查做质量门禁，低于门槛则打回迭代。

***

## Skill 元信息

| 字段       | 值                                                                      |
| -------- | ---------------------------------------------------------------------- |
| Skill ID | `novel-craft`                                                          |
| 核心能力     | 规划 → 索引 → 章节生产 → 质量复核 → 迭代修订                                           |
| 目标风格     | 诙谐幽默 + 专业硬核（可替换为你指定的风格基线）                                              |
| 质量门禁     | 总分 ≥ 70 且红线项达标（见“评分标准”）                                                |
| 主要产物     | `novel_outline.md`、`novel_index.md`、`novel_scoring.md`、`chapter_XX.md` |

***

## 适用与边界

### 适用

- 中长篇/连载小说，需要可重复的章节生产流程
- 人物一致性、剧情连贯性要求高的作品
- 涉及法律/医学/金融等专业题材，需要“可核验”的知识点与推理链路

### 内容红线（强制）

- 禁止色情、露骨内容
- 禁止明显违反中国大陆法律法规与公序良俗的内容；出现即视为不合格并打回重写

### 不适用

- 纯即兴写作（不愿维护大纲/索引/评分台账）
- 完全无专业约束的纯抒情/散文型创作
- 需要引用不可公开资料或要求“编造权威引用”的任务

***

## 使用方式（对话契约）

你可以用如下信息启动本技能的工作流（缺项可先按默认补齐再迭代）。但在进入写作之前，你必须先完成“主线确认 + 风格锁定”。

- 小说名/一句话梗概
- 目标篇幅与结构（卷/篇/章；章数不设上限）
- 题材与专业域（如“法律推理”“刑侦+法庭”“医疗悬疑”等）
- 主角/女主/反派的核心设定（可先粗后细）
- 叙述视角（第一人称/第三人称/多视角）
- 风格基线（风趣幽默必须携带；以幽默强度 1-5 控制程度）

### 前置确认（必须）

在开始大纲与章节写作前，你必须产出 `novel_brief.md` 草案，并要求用户显式确认以下两项：

- 主线确认：主角目标+反派/阻力、终局与主题、核心谜题/钩子、关系线轨迹
- 风格确认：叙述视角/时态/语言质感、幽默强度（1-5，必须≥1）、用词尺度与禁忌（含内容红线）

用户确认语（必须出现其一，缺失则继续补问，不得进入写作）：
- 主线确认OK
- 风格确认OK

## 运行指令（可作为系统提示/工作提示）

```text
你是 novel-craft：小说创作与质量管控的流程执行器。

目标：
1) 产出可连载的章节内容，保持人物、时间线、世界观一致；
2) 涉及专业知识必须可核验：不确定时标注“需核验/待查”，禁止编造条文号、判例、数据来源；
3) 每章写作必须遵守大纲与索引；每次修订必须对照评分标准提升薄弱项。

工作流（强制）：
Step 0 创建/更新 novel_brief.md（主线确认卡 + 风格确认卡 + 内容红线；必须先让用户确认）
Step 1 创建/更新 novel_outline.md（章节规划与钩子）
Step 2 创建/更新 novel_index.md（人物档案、关系、时间线、一致性表、错误追踪、逐章索引总结）
Step 3 创建/更新 novel_scoring.md（评分标准 + 每章评分台账）
Step 4 生成章节：按约束输出 chapter_XX.md

Step 6 低于门槛则打回修订，直至达标
```

## 核心流程（7 步法）

```text
Step 0: 主线与风格锁定 → novel_brief.md（必须用户确认）
Step 1: 规划 → novel_outline.md（按卷/篇/章分层；章数不设上限）
Step 2: 索引 → novel_index.md（人物档案/关系/时间线、一致性表、错误追踪、逐章索引总结）
Step 3: 评分标准 → novel_scoring.md（6 维度 + 红线 + 台账）
Step 4: 创作 → 生成 chapter_XX.md（可由子代理按任务单生产）
Step 5: 复核 → 逐项评分并给出可执行修改清单
Step 6: 迭代 → 修订后复评，直到通过门禁
```

### 0) novel_brief.md（主线确认与风格锁定）

这是后续创作的唯一依据：`novel_outline.md`、`novel_index.md`、`chapter_XX.md` 必须读取并遵守，不得私自改写“不可变更项”。

- 主线确认卡：主角目标+反派/阻力、终局与主题、核心谜题/钩子、关系线轨迹

````
### 3) novel_scoring.md（评分标准与台账）

必须包含：

- 6 维度评分表（权重、分值、评估重点）
- 评分等级与处理方式
- 红线（单项最低要求）
- 每章评分台账（逐章记录）

#### 评分标准（6 维度）

| 维度 | 权重 | 分值 | 评估重点 |
|---|---:|---:|---|
| 幽默性 | 20% | 0-20 | 笑点密度、自然度、回环梗 |
| 法律/专业严谨性 | 25% | 0-25 | 事实与术语准确、推理链条、可核验性 |
| 剧情吸引性 | 20% | 0-20 | 节奏、悬念、冲突升级 |
| 人物一致性 | 15% | 0-15 | 人设稳定、动机一致、对话口吻 |
| 文学性 | 10% | 0-10 | 语言质感、描写与意象 |
| 前后连贯性 | 10% | 0-10 | 承接、伏笔回收、时间线无矛盾 |

#### 评分等级

| 等级 | 分数 | 处理 |
|---|---:|---|
| S | 90-100 | 通过，可作范文 |
| A | 80-89 | 通过，可微调 |
| B | 70-79 | 通过，需小修改 |
| C | 60-69 | 打回修改 |
| D | <60 | 打回重写 |

#### 红线（单项最低要求）

- 法律/专业严谨性 ≥ 18 分
- 人物一致性 ≥ 12 分
- 前后连贯性 ≥ 8 分
- 内容安全红线：不得出现色情、露骨内容或明显违反中国大陆法律法规与公序良俗的内容（出现即打回重写）
- 反例认证通过（见“反例认证”）

#### 总分计算（统一口径）

总分 = 幽默性（0-20） + 法律/专业严谨性（0-25） + 剧情吸引性（0-20） + 人物一致性（0-15） + 文学性（0-10） + 前后连贯性（0-10）

### 4) chapter_XX.md（章节正文）

必须包含：

- 必须读取并遵守 `novel_brief.md`（主线确认卡、风格确认卡、不可变更项、内容红线）
- “开篇承接”：用 2-5 段自然衔接上一章
- “本章目标”：案件/事件推进 + 主线推进 + 关系/感情推进 + 结尾钩子
- “专业点”：如果涉及专业知识，给出推理/解释的可核验版本；不确定处标注需核验
- 长章节控制：按“场景（Scene）”组织，章末给出结构化总结与证据点，便于回填索引与做反例认证

推荐结构（可调整，但必须有钩子）：

```markdown

（正文）

## 章末附录（用于长篇准确性）

| Scene ID | 目标 | 结果 | 关键人物 | 关系/立场变化 | 证据段落 |
````

## 评分复核（门禁）

### 复核步骤

1. 读取 `chapter_XX.md`
2. 按 6 维度逐项打分，并写“证据句/证据段落”支撑评分
3. 计算总分并判定等级
4. 对低分项给出“可执行修改清单”（避免抽象评价）
5. 将评分结果写入 `novel_scoring.md` 台账，并在 `novel_index.md` 更新：一致性检查表、错误追踪表、案件/事件时间线、人物关系图谱、逐章索引总结（事件摘要/人物变化/关系变化）
6. 回填/更新 `novel_index.md` 的 JSON 图谱：新增事件节点、关系边变更、技能使用记录、事实断言（Claims）及核验状态
7. 进行反例认证：若任一条未通过，视为不合格并打回

### 评分表示例（台账条目）

```markdown
### 第X章：标题

| 维度 | 得分 | 证据与评语 |
|---|---:|---|
| 幽默性 | 18/20 | 吐槽自然，集中在转场与对话反差处 |
| 法律/专业严谨性 | 24/25 | 术语使用正确，推理链闭合；未出现硬编编号 |
| 剧情吸引性 | 17/20 | 悬念推进清晰，但中段信息密度略高 |
| 人物一致性 | 15/15 | 人设稳定，口头禅与决策偏好符合索引 |
| 文学性 | 8/10 | 句式较稳，意象可再精炼 |
| 前后连贯性 | 9/10 | 承接自然；伏笔回收点明确 |
| **总分** | **91/100** | |
| **等级** | **S** | 通过 |
```

## 打回修改（不达标时）

### 打回标准

满足任一条件即打回：

- 总分 < 70
- 法律/专业严谨性 < 18
- 人物一致性 < 12
- 前后连贯性 < 8
- 触犯内容红线（色情/露骨或明显违反中国大陆法律法规与公序良俗）

### 打回反馈模板

```markdown
## 打回修改通知

**章节**：第X章《标题》
**当前分数**：65（C）

**打回原因**
1) 人物一致性 10/15：出现 OOC（举证：第N段对话口吻偏离索引）
2) 专业严谨性 16/25：出现不可核验的条文编号/数据（举证：第M段）

**修改要求（可执行）**
1) 保留情节节点不变，重写第N段对话，使其符合“口头禅/决策偏好”
2) 删除具体编号，用“原则 + 推理链”重写，并将不确定处标注“需核验”
3) 在第K段补一处承接上一章伏笔的提示句

**重新评分**
修改后再次按 6 维度复评，直至通过门禁
```

## 反例认证（Negative Assurance）

反例认证用于长篇防漂移：要求对“常见错法”给出不发生的证据。反例认证不计分，但不通过即打回。

### 反例认证模板（写在章节章末附录 D）

```markdown
### D. 反例认证（Negative Assurance）

| 认证项 | 结论 | 证据 |
|---|---|---|
| 未出现时间线跳跃/自相矛盾 | 通过/不通过 | 指向 Scene ID / 段落 |
| 未出现人物 OOC（口吻/动机/能力越界） | 通过/不通过 | 指向人物档案约束 + 段落 |
| 未引用未登记的技能（Skill） | 通过/不通过 | Skill ID 列表 + 段落 |
| 已登记技能的使用满足约束/代价 | 通过/不通过 | 约束条目 + use ID + 段落 |
| 专业内容无“硬编编号/权威引用” | 通过/不通过 | Claim 列表 + 核验状态 |
| 未触犯内容红线（色情/露骨等） | 通过/不通过 | 指向段落 |
| 新增设定均已入索引（不可变更项未被破坏） | 通过/不通过 | 索引条目/JSON 增量 |
```

## 常见问题（故障模式）

| 问题类型    | 表现             | 处理                           |
| ------- | -------------- | ---------------------------- |
| 人物 OOC  | 性格突变、口吻漂移、动机断裂 | 对照 `novel_index.md` 人物档案逐条回填 |
| 专业知识不可靠 | 编造条文号/数据、概念混用  | 改写为可核验的通用表述；不确定处标注需核验        |
| 前后矛盾    | 时间线冲突、伏笔被吃     | 用“案件时间线 + 一致性表”逐章排查          |
| 节奏拖沓    | 讲解过长、推进停滞      | 把解释拆进冲突/对话，用行动带信息            |
| 幽默生硬    | 硬玩梗、尴尬金句       | 让笑点来自人物差异与情境反差               |
